H.AI · History AI

Gen AI 기반 학생 참여형 역사 디지털 교과서 프로젝트입니다. 실시간 상호작용과 AI 응답 경험을 결합해, 기존의 일방향 학습보다 더 적극적인 참여를 유도하는 서비스를 목표로 했습니다.

2025.11 – 2026.01 7인 팀 프로젝트 백엔드 담당 최종 프로젝트 대상

프로젝트 개요

  • 문제: 텍스트 중심·정적 시각자료·단방향 수업으로 인해 학습 참여도와 몰입도가 낮은 구조
  • 목표: AI와 시각적 인터랙션을 결합해 학생 참여를 유도하는 Gen AI 기반 디지털 역사 교과서 구현
  • 해결 방향: “읽는 역사”가 아닌 대화하고 탐색하는 학습 경험으로 전환

핵심 기능

  • 인증 및 권한 처리 · JWT 기반 로그인과 사용자 요청 제어
  • 실시간 채팅 · WebSocket 기반 메시지 송수신 처리
  • AI 응답 처리 · Django에서 Bedrock 호출 후 응답 반환
  • 데이터 관리 · PostgreSQL 관계 설계와 Redis 캐싱 적용

트러블슈팅

1. WebSocket 환경에서 인증 구조 개선

문제
  • STOMP 메시지 Payload의 userId에 의존하는 구조
  • userId를 임의 조작할 경우 권한 도용 및 데이터 부정 접근이 가능한 보안 취약점 존재
  • 학급·역할 기반 권한 검증 기능 특성상, 인증 신뢰가 깨지면 권한 통제 불가능
원인
  • WebSocket은 HTTP Handshake 이후 별도의 프로토콜로 동작하여 JWT 필터 체인이 적용되지 않음
  • STOMP 메시지 처리 과정에서는 인증 정보가 자동으로 전달·유지되지 않음
해결
  • ChannelInterceptor로 CONNECT 프레임 내 토큰 검증
  • Authentication 객체를 생성해 Principal로 설정
  • 세션 기반으로 이후 메시지까지 인증 유지
결과
  • 클라이언트 조작 기반 사용자 위장 및 무단 접근 차단
  • WebSocket 환경에서도 HTTP와 동일한 인증 체계 확보
  • 인증 로직을 서버 중심으로 통합하여 향후 기능 확장에도 유연하게 대응 가능

2. Redis Timeout 발생 원인 추적 및 TLS 설정 불일치 해결

문제
  • ECS 배포 후 AI 요약 기능 호출 시, 에러 로그 없이 요청이 중단되는 현상 발생
  • Redis 기반의 핵심 기능 불능으로 인해 서비스 전체의 사용자 흐름 단절
원인 추적
  • 로그가 전무한 점에 착안, 애플리케이션 도달 전 '연결 단계'의 이슈로 판단
  • ECS 컨테이너 내부에서 직접 점검 수행
  • TCP 연결 테스트 → 정상 (네트워크 문제 배제)
  • redis-py PING 테스트 → timeout 발생
  • 연결은 되지만 응답이 없는 상태 확인
원인
    ElastiCache Redis는 TLS 연결을 요구 애플리케이션은 평문(redis://)으로 접속 TCP 연결은 성공하지만 실제 요청은 처리되지 않아 timeout 발생했음을 확인
해결
  • Redis 연결을 rediss:// 기반으로 변경 및 SSL 옵션 적용
  • Redis.from_url()로 환경별 설정 통합
결과
  • Redis timeout 문제 해결 및 서비스 정상화
  • 인프라 설정(TLS) 중요성을 파악하여 환경별 설정 차이로 인한 장애 재발 방지
